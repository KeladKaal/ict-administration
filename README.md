# Администрирование в инфокоммуникационных технологиях

> 🌐 **Язык:** Русский (ветка `main-rus`) · [English version → branch `main`](https://github.com/KeladKaal/ict-administration/tree/main)

Практический курс для тех, кто только начинает в devops.

Разбираем системы, которые devops настраивает и поддерживает, чтобы поверх них работали разработчики — базы данных, брокеры сообщений, поиск, хранилища, мониторинг и не только. Идея простая: **devops настраивает — разработчик пользуется.** К концу курса ты будешь понимать, что это за системы, зачем они разработчику и что нужно, чтобы их нормально эксплуатировать (доступы, бэкапы, мониторинг и типичные проблемы).

Глубоких знаний заранее не нужно — достаточно базовой командной строки и Docker.

## Лекции

Семь лекций по 1,5 часа. Каждая — три коротких блока (~25–30 мин). Кликни по лекции, чтобы посмотреть содержание.

1. [Введение, роль devops, GitLab + CI/CD](lecture-1-introduction/) — о чём курс и как код попадает на прод
2. [Базы данных](lecture-2-databases/) — PostgreSQL/MySQL: доступы, репликация, бэкапы, миграции
3. [Брокеры сообщений и кэш](lecture-3-brokers-cache/) — Kafka, RabbitMQ, Redis
4. [Поиск и логирование](lecture-4-search-logging/) — Elasticsearch / ELK
5. [Хранилище, доставка и безопасность](lecture-5-storage-delivery-security/) — S3, CDN, nginx, секреты
6. [Мониторинг и observability](lecture-6-monitoring-observability/) — Prometheus, Grafana, метрики/логи/трейсы
7. [AI-инфраструктура](lecture-7-ai-infrastructure/) — агенты, инструменты, MCP, n8n, хостинг и стоимость

## Использование AI в лабах

Пользоваться AI-ассистентом можно и нужно — но репозиторий настроен так, чтобы он **помогал учиться, а не выдавал готовые ответы**: работал поэтапно, объяснял причины и проверял понимание перед переходом дальше.

Правила лежат в [`AGENTS.md`](AGENTS.md) (и в `CLAUDE.md` для Claude Code). Большинство AI-инструментов подхватывают их сами. **Если твой инструмент не подхватил — просто укажи ему на [`AGENTS.md`](AGENTS.md)** и попроси следовать этому файлу.

## Как устроен репозиторий

- Одна папка на лекцию, внутри — короткий README с описанием блоков.
- **`main`** — всё на **английском**. **`main-rus`** — то же на **русском**.
- Всё, что сюда пушится, дублируется на двух языках: английский → `main`, русский → `main-rus`.
