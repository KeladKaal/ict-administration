# Lecture 3 — Message brokers and cache

How services talk to each other without blocking, and how to make things fast with a cache.

**Block 1 — Why brokers.** Synchronous vs asynchronous communication, queues vs streams. RabbitMQ: exchanges, queues, routing.

**Block 2 — Kafka.** Topics, partitions, consumer groups. How it differs from RabbitMQ and when to pick which.

**Block 3 — Redis.** Cache and broker in one: data structures, TTL, persistence, common problems (eviction, hot keys).
